## Overview
AIHub is a full-stack, role-aware platform built to operationalize end-to-end hardware workflows for course teaching at Tsinghua University iCenter. The system consolidates **hardware cataloging**, **borrow requests**, **purchase requests**, **approval workflows**, **supplier records**, and **course-level analytics** into a single internal web application that can be used by instructors and teaching staff across multiple courses. It follows a modern, maintainable architecture: a **React + TypeScript (Vite)** front end for interactive dashboards and admin tooling, a **Flask REST API** backend, and a **MySQL** database schema designed to support users/roles, courses/participants, hardware inventory, rental/borrow history, suppliers, and AI interaction logs. The platform also includes an **AI Chat assistant** that retrieves relevant database context (hardware/course/user records) and then calls an LLM to produce grounded answers—turning the system from a traditional admin panel into an “ask-and-operate” workspace.

**Delivery outcome:** the system was successfully delivered to course instructors and adopted for internal teaching operations at iCenter, supporting hardware-related processes across multiple courses.

## My Contributions
- **System architecture & implementation (end-to-end):** Designed and implemented the full-stack pipeline (**React/TypeScript/Vite → Flask API → MySQL**), including API routing, CRUD services, and a schema aligned with real operational entities (users/roles, courses, participants, inventory, requests, approvals, suppliers, history, AI logs).
- **Role-aware product experience:** Built protected routing and role-based access patterns for common internal roles (e.g., students/requesters, TAs/admins, instructors/approvers), ensuring that each role sees the correct operations and data surfaces.
- **Core business modules delivered to production use:**
  - Hardware catalog management (create/update/list/detail; filtering/search)
  - Borrow/rental request submission, status tracking, and history
  - Purchase request submission and approval pipeline
  - Course management and course-level usage statistics dashboards
  - Supplier and procurement-related records management
- **AI assistant with database grounding:** Implemented a workflow for **intent routing → database retrieval → LLM response generation**, enabling natural-language queries such as “available GPUs,” “borrow status,” “course usage summaries,” and “who requested device X,” while keeping answers grounded in the system’s database state.
- **Engineering quality & maintainability:** Established a scalable front-end structure (modular pages/components, typed models, predictable state management), and implemented backend safeguards (CORS configuration, password hashing, parameterized queries) with clear improvement hooks (environment-variable secrets management, API consistency checks).

## Methodology Pipeline
**Requirements discovery (iCenter teaching hardware workflows) → data model & schema design (MySQL) → REST API design/implementation (Flask) → front-end dashboard & workflows (React + TypeScript + Vite) → role-aware routing & state management → AI Chat (DB retrieval + LLM calling) → integration testing & instructor feedback loop → internal delivery and adoption across multiple courses**
